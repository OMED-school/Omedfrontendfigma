�� Deployment Guide

Learn how to deploy OMED to various platforms.

Web Deployment

Build for Production

npm run build

This creates optimized static files in the /build directory.

Deploy to Vercel

  1. Install Vercel CLI: npm i -g vercel
  2. Run: vercel
  3. Follow prompts to deploy

Deploy to Netlify

  1. Install Netlify CLI: npm i -g netlify-cli
  2. Run: netlify deploy --prod
  3. Point to /build directory

Deploy to GitHub Pages

  1. Build the app: npm run build
  2. Create gh-pages branch
  3. Copy build files to gh-pages
  4. Push to GitHub
  5. Enable GitHub Pages in repository settings

Mobile Deployment

iOS Deployment

  1. Open Xcode project: npx cap open ios
  2. Configure signing & capabilities
  3. Archive the app: Product → Archive
  4. Upload to App Store Connect
  5. Submit for review

Android Deployment

  1. Open Android Studio: npx cap open android
  2. Build → Generate Signed Bundle / APK
  3. Upload to Google Play Console
  4. Create release and submit

Environment Configuration

Set up environment variables for different environments:

# .env.production VITE_SUPABASE_URL=https://your-project.supabase.co VITE_SUPABASE_ANON_KEY=your-production-key VITE_APP_ENV=production

Continuous Deployment

Set up CI/CD with GitHub Actions to automatically deploy on push to main branch.

Production Checklist

  • ✅ Environment variables configured
  • ✅ Supabase RLS policies enabled
  • ✅ Authentication configured
  • ✅ Database backups enabled
  • ✅ SSL/HTTPS enabled
  • ✅ Error tracking configured
  • ✅ Analytics set up
  • ✅ Performance monitoring active
